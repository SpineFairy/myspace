# 예지 제어 모델 만들기

## 학습
- 강화학습으로 예지 제어 모델을 만들 것임
- 만들어놓은 탐지분류모델(unified_model (4).pkl), 탐지분류모델 코드(IFRF.py)
    - 탐지분류모델은 4개의 센서 값(입력) 온도, 습도, 진동, 레일 속도를 받아서 이상을 탐지 및 타입 분류를 함
- 결과(분류라벨)을 가지고 최적의 제어 행동을 하는 예지 제어 모델이 목표
- 예지 제어 모델은 출력으로 어떤 제어를 몇단으로 했는지 출력되어야함.  

## 조건
전제가 탐지분류에서의 값이 예지 모델로 들어오는 것임.  
즉, 이상인 값만 들어온거고 예지는 값을 정상으로 돌리기만 하면 된다.  
예지가 해당 값이 어떤 상태인지 알 필요는 없다.  
그건 탐지분류에서 하는 일이지 이미 들어온 값 자체가 이상한거다.

## 세부사항
- 점수는 정수로 계산할 것이고, 레일 속도(단계) 표시도 당연히 정수, 온도랑 습도랑 진동은 소숫점 3자리 까지만 계산할 것임.
- 특별한 변수 커플링은 없음, RL이 모든 액추에이터를 쓸 수 있게 열어두고 라벨은 가이드로 줄것임,
- observation으로는 센서 값(온도, 습도, 진동, 레일 속도(단수)), 직전 제어 값(히터, 쿨링, 제습기, 레일 기어 단수), 분류 라벨
- 초기 센서 값은 정상 밴드에서 벗어난 값으로 설정
- 1 step당 1초 (t)

## 강화학습의 환경(env) 변수

환경은 Gym 스타일로 작성

### 분류 라벨 및 제어 행동
```
온도 급상승 -> TEMP_RUNAWAY -> 쿨링  
습도 상승, 온도 하락 -> HUM_UP_TEMP_DOWN -> 제습기, 히터  
진동 상승, 온도 상승 -> VIB_RISE_TEMP_LAG -> 레일 속도 조절, 쿨링  
진동 스파이크 -> VIB_SPIKE_REPEAT -> 레일 속도 조절 or stop  

쿨링(0~3): 온도를 감소  
제습기(0~3): 습도를 감소  
히터(0~3): 온도를 상승  
레일 단수(1~6): 단수에 따라 진동 상승 or 감소  
```

### 정상 밴드
```  
온도 27.000 ~ 33.000,    
진동 2.000 ~ 4.000,    
습도 40.000 ~ 50.000  
```

### 위험 한계
```
온도 (5, 75)  
습도: (5, 80)  
진동: (0.5, 8)   
```

### 행동 수치  
```
제습기(값 감소), 쿨링(값 감소), 히터(값 증가)  
1단 1% ~ 20%   
2단 21% ~ 40%   
3단 41% ~ 60%  

레일 단수 - 진동 변화  
1단 -20% ~ 30%  
2단 -10% ~ -20%  
3단 0  
4단 +1% ~ 10%  
5단 +11% ~ 20%  
6단 +21% ~ 30%  
```

### Observation Space
연속값(Box):
- temp, hum, vib는 float (소수점 3자리)
- rail gear는 integer

Action Space
- MultiDiscrete([4, 4, 4, 6])
